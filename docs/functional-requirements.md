# 機能要件

## お問い合わせ管理アプリ（学習用）

[← 要件定義書に戻る](requirements.md)

---

## 1. 共通機能

### F-01 役割トグル

| 項目 | 内容 |
|------|------|
| 概要 | 画面上部のトグルスイッチで「ユーザー」「担当者」を切り替える |
| 表示位置 | 全画面のヘッダー右側 |
| 初期状態 | ユーザーモード |
| 切り替え後の動作 | 即座に画面表示・操作可能な機能が切り替わる |

---

## 2. ユーザーモードの機能

### F-02 チケット作成

| 項目 | 内容 |
|------|------|
| 概要 | タイトル・本文・優先度を入力して新規お問い合わせを作成する |
| 入力項目 | タイトル（必須）・本文（必須）・優先度（必須、デフォルト: 中） |
| 作成後の動作 | チケット一覧画面へ遷移する |

### F-03 チケット一覧（自分のみ）

| 項目 | 内容 |
|------|------|
| 概要 | 自分が作成したチケットのみを一覧表示する |
| 表示項目 | ID・タイトル・ステータス・優先度・作成日時 |
| 並び順 | 作成日時の降順（新しい順） |

### F-04 チケット詳細表示（ユーザーモード）

| 項目 | 内容 |
|------|------|
| 概要 | チケットの内容・ステータス・コメント履歴を確認する |
| 表示項目 | タイトル・本文・優先度・ステータス（表示のみ）・作成日時・更新日時・コメント一覧 |
| 操作制限 | ステータスの変更は不可 |

### F-05 コメント追加（ユーザー）

| 項目 | 内容 |
|------|------|
| 概要 | 自分のチケットにコメント（追記）を投稿する |
| 投稿者ラベル | `ユーザー` |
| 入力項目 | コメント本文（必須） |
| 投稿後の動作 | コメント欄をクリアし、一覧の末尾に追加表示する |

---

## 3. 担当者モードの機能

### F-06 チケット一覧（全件）

| 項目 | 内容 |
|------|------|
| 概要 | 全ユーザーのチケットを一覧表示する |
| 表示項目 | ID・タイトル・ステータス・優先度・作成日時 |
| 並び順 | 作成日時の降順（新しい順） |

### F-07 チケット詳細表示（担当者モード）

| 項目 | 内容 |
|------|------|
| 概要 | チケットの内容・ステータス・コメント履歴を確認する |
| 表示項目 | タイトル・本文・優先度・ステータス（変更可）・作成日時・更新日時・コメント一覧 |

### F-08 ステータス変更

| 項目 | 内容 |
|------|------|
| 概要 | チケットのステータスを変更する |
| 操作方法 | チケット詳細画面のドロップダウンから選択 |
| 選択肢 | `未対応` / `対応中` / `解決済み` |
| 変更後の動作 | 即座に保存し、更新日時を更新する |

### F-09 コメント追加（担当者）

| 項目 | 内容 |
|------|------|
| 概要 | チケットに対応コメント（返信）を投稿する |
| 投稿者ラベル | `担当者` |
| 入力項目 | コメント本文（必須） |
| 投稿後の動作 | コメント欄をクリアし、一覧の末尾に追加表示する |
