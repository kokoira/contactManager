# 機能要件

## お問い合わせ管理アプリ（学習用）

[← 要件定義書に戻る](requirements.md)

---

## 1. 共通機能

### F-01 役割トグル

| 項目 | 内容 |
|------|------|
| 概要 | 画面上部のトグルスイッチで「ユーザー」「担当者」を切り替える |
| 表示位置 | 全画面のヘッダー右側 |
| 初期状態 | ユーザーモード |
| 切り替え後の動作 | 即座に画面表示・操作可能な機能が切り替わる |

---

## 2. ユーザーモードの機能

### F-02 チケット作成

| 項目 | 内容 |
|------|------|
| 概要 | タイトル・本文・緊急度を入力して新規お問い合わせを作成する |
| 入力項目 | タイトル（必須）・本文（必須）・緊急度（必須、デフォルト: 中） |
| 作成後の動作 | チケット一覧画面へ遷移する |

### F-03 チケット一覧（ユーザーモード）

| 項目 | 内容 |
|------|------|
| 概要 | チケットを一覧表示する |
| 表示項目 | ID・タイトル・ステータス・緊急度・作成日時 |
| 並び順 | 作成日時の降順（新しい順） |
| 備考 | 削除されていないチケットを全件表示する。削除済みチケットは担当者モードのみ表示される |

### F-04 チケット詳細表示（ユーザーモード）

| 項目 | 内容 |
|------|------|
| 概要 | チケットの内容・ステータス・コメント履歴を確認する |
| 表示項目 | タイトル・本文・緊急度・ステータス（表示のみ）・作成日時・更新日時・コメント一覧 |
| 操作制限 | ステータスの変更は不可（表示のみ）。緊急度は変更可 |

### F-05 コメント追加（ユーザー）

| 項目 | 内容 |
|------|------|
| 概要 | 自分のチケットにコメント（追記）を投稿する |
| 投稿者ラベル | `ユーザー` |
| 入力項目 | コメント本文（必須） |
| 投稿後の動作 | コメント欄をクリアし、一覧の末尾に追加表示する |

---

## 3. 担当者モードの機能

### F-06 チケット一覧（全件）

| 項目 | 内容 |
|------|------|
| 概要 | 全ユーザーのチケットを一覧表示する |
| 表示項目 | ID・タイトル・ステータス・緊急度・作成日時 |
| 並び順 | 作成日時の降順（新しい順） |
| 備考 | 削除済みチケットも含めて表示する（薄いスタイルで表示） |

### F-07 チケット詳細表示（担当者モード）

| 項目 | 内容 |
|------|------|
| 概要 | チケットの内容・ステータス・コメント履歴を確認する |
| 表示項目 | タイトル・本文・緊急度・ステータス（変更可）・作成日時・更新日時・コメント一覧 |

### F-08 ステータス変更

| 項目 | 内容 |
|------|------|
| 概要 | チケットのステータスを変更する |
| 操作方法 | チケット詳細画面のドロップダウンから選択 |
| 選択肢 | `未対応` / `対応中` / `解決済み` |
| 変更後の動作 | 画面上部の「未保存の変更があります」バナーから「保存」ボタンを押下して保存する。保存後に更新日時を更新する |

### F-09 コメント追加（担当者）

| 項目 | 内容 |
|------|------|
| 概要 | チケットに対応コメント（返信）を投稿する |
| 投稿者ラベル | `担当者` |
| 入力項目 | コメント本文（必須） |
| 投稿後の動作 | コメント欄をクリアし、一覧の末尾に追加表示する |

---

## 4. ユーザーモードの追加機能

### F-10 チケット削除（ソフトデリート）

| 項目 | 内容 |
|------|------|
| 概要 | ユーザーがチケットを論理削除する |
| 対象ロール | ユーザーのみ（担当者は操作不可） |
| 操作方法 | チケット詳細画面の「削除する」ボタン。確認ダイアログで承認後に実行 |
| 削除後の動作 | チケット一覧画面へ遷移する |
| 削除済みチケットの表示 | ユーザー：一覧に非表示。担当者：一覧に薄いスタイルで表示。詳細画面では削除済みバナーを表示し、ステータス変更ドロップダウンと削除ボタンを非表示にする。緊急度はバッジ表示（変更不可）。メッセージ投稿は引き続き可能 |
| 備考 | 物理削除ではなく `deleted_at` にタイムスタンプを設定するソフトデリート |
